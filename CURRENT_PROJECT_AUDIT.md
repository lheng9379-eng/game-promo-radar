# 当前项目审计

审计时间：2026-07-11

## 当前已有功能

- Streamlit 本地页面，固定运行在 `http://localhost:8503/`。
- DuckDB 本地数据库：`data/game_promo_radar.duckdb`。
- 旧版 `tasks` 任务表、任务备注、采集日志、扩展情报表、历史结算表。
- 公开网页采集：入口页抓取、候选链接抽取、详情页解析、HTML 快照保存、图片保存、可选 OCR。
- 种子详情页采集：TapTap、B站等固定种子。
- DuckDuckGo 关键词搜索采集。
- 手动链接、Excel 导入导出。
- 基础可做性评分、制作难度判断、截止提醒、采集日志和自动采集配置。
- Playwright 持久化浏览器登录读取函数，但尚未形成完整 UI 工作流。

## 实际可用功能

- 本地数据库初始化和旧表迁移可用。
- 手动任务保存、Excel 导入导出可用。
- 公开 HTML 快照保存与重解析可用。
- 对结构化或半结构化活动页可提取奖励、门槛、截止、素材和制作要求。
- 页面来源链接使用 `LinkColumn` 或 `link_button` 可点击。
- 测试覆盖当前核心逻辑，升级后 `pytest` 为 74 passed。

## 无效或占位功能

- 多个平台在旧 `config/sources.yaml` 中仅为 placeholder，不应视为可采集能力。
- `MediaCrawlerAdapter` 热度分析尚未配置，页面会提示未获取数据。
- 旧自动采集只在 Streamlit 运行期间触发，不是系统级后台任务。
- 登录平台仅有基础 Playwright 函数，未做验证码绕过，也不应自动绕过。
- 旧 `tasks` 表混入平台首页、说明页、开发者文档页，不能等同于有效推广活动。

## 采集失败原因

- 部署成功只代表 Streamlit 服务启动，不代表自动采集已启用；当前 `data/auto_collect_config.json` 中 `enabled=false`。
- 旧公开采集从入口页开始，入口页经常缺少奖励、门槛、报名入口、截止时间，被详情阈值过滤。
- 搜索依赖 DuckDuckGo HTML 结果，稳定性受网络、搜索结果质量和反爬策略影响。
- 当前数据库 27 条旧任务中，多条是平台入口页或说明页，字段完整度低。
- 历史日志显示 Streamlit 曾因重复 checkbox 自动 ID 崩溃；当前相关控件已使用唯一 key。

## 数据库问题

- 旧 `tasks` 表不能区分 `content_platform` 与 `source_platform`。
- 旧结构没有 `campaign_candidates` 与 `campaigns` 分层，导致线索和正式任务容易混在一起。
- 缺少来源可信度、风险信号、候选校验、实际收益回填与评分校正结构。
- 旧 `settlements` 表存在但未形成完整任务闭环。
- 已新增：`data_sources`、`campaign_candidates`、`campaigns`、`campaign_progress`。

## 页面逻辑问题

- 首页此前以旧任务为中心，无法突出候选商机、正式商机、数据源状态和采集失败。
- 候选任务与正式任务此前没有视觉和数据层隔离。
- 旧“数据源”页展示旧配置，无法表达登录要求、可信等级和连续失败状态。
- 页面导出以旧任务表为主，正式商机导出需从新商机页执行表格下载或后续补专门按钮。

## 建议删除、保留和重构

- 保留：DuckDB、本地 Streamlit、公开页面快照、手动导入、Excel 导入导出、旧 `tasks` 兼容层。
- 保留但降级：旧固定平台配置，只作为旧版兼容入口。
- 删除或停止扩展：没有真实采集能力的 placeholder 适配器。
- 重构：采集入口统一先写入 `campaign_candidates`，通过校验后才进入 `campaigns`。
- 重构：评分模型以实际收益确定性、账号匹配、制作时长、素材、剩余时间和来源可信度为核心，不再用总奖池代表预期收益。
